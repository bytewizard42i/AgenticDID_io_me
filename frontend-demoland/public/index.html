Privacy-preserving delegated authority for autonomous agents — three-pillar model on Midnight

🏛️ The three pillars ⓘDIDz root identity + AgenticDID agent authority + RWAz asset ownership — the foundational layers everything else builds on

DIDz Root

Universal identity layer. Permanent registry entries, lifecycle status, entity types. machine 1

AgenticDID

Agent identity + delegated authority. Scoped grants, attenuation, ZK proof of authority. machine 2

RWAz

Object/RWA identity. VIN stays, title moves. Encumbrances, provenance, fractionalization.

⚙️ Two machines ⓘMachine 1 = identity (who you are). Machine 2 = permission (what you can do). Never confuse the two.

🔐 Machine 1: Identity

The agent's DIDz — a permanent, non-transferable registry entry with entity type AgentDIDz and lifecycle status.

Contract: DIDzRegistry.compact (15 circuits)

Key circuits: register_did, assert_i_control, prove_entity_type, rotate_owner_key, set_terminal_status

Privacy: On-chain = existence + status only. Entity type, subject binding, owner key = committed, revealed via ZK.

🎫 Machine 2: Permission

The scoped-grant system — bounded, revocable, attenuating authority delegated from principal to agent.

Module: midnight-modules/scoped-grant v2

Key circuits: issue_grant, delegate, assert_authorized, revoke_grant, advance_epoch

ZK proof: Verifier learns one bit — "authorized for this scope and amount, unexpired, unrevoked" — nothing else.

📜 Constitution alignment

This demo follows the DIDz Constitution (Appendix A: AgenticDID rules). Key invariants enforced:

  • A2: Two machines — identity and permission are never confused
  • A4: Attenuation-only — child grants are always strict subsets of parents
  • §17: Dual spend limits — per_action_cap AND cumulative_cap, both enforced
  • §20: Cascade revocation — revoking a root invalidates the entire subtree
  • §22: ZK proof of authority — verifier learns one bit, nothing else
  • §27: Constitutional violations trigger grant revocation, not identity deletion
🧠 What this demonstrates
Scoped grants replace broad API keys with narrow, expiring, revocable authority — "buy groceries up to $50" instead of "access my bank account."

Attenuation means an agent can delegate to a sub-agent, but only with less power. The delegation tree is a strict subset chain.

ZK proof of authority lets a verifier confirm "this agent is authorized for this action" without learning who the principal is, what other grants exist, or the delegation topology.

Cascade revocation kills an entire delegation subtree in one action — revoke the root, every child grant dies instantly.

Constitutional enforcement means violations (exceeding authority, impersonating, bypassing revocation) trigger grant revocation and status changes — but the permanent identity record is never deleted.
